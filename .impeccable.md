# Design Context

## Users

Developers, IT and security professionals, and curious technical users who encounter a suspicious file (an email attachment, a download, a file dropped by a process) and need to quickly decide: **is this dangerous, and what does it mean?** They are often mid-investigation, at a laptop, with some urgency but not panic.

The secondary audience is the CloudsineAI review team evaluating this take-home. They will judge on **functionality, code quality, problem-solving, creativity, and presentation** (from `docs/assignment.md`). Presentation is an explicit criterion — the interface must feel *crafted*, not templated.

## Brand Personality

**Three words: precise, calm, credible.**

- **Precise** — this is a measurement instrument. Numbers, hashes, detection counts must feel crisp. Typography should be exacting. Hierarchy should be unambiguous.
- **Calm** — security work is stressful. The UI reduces anxiety, it does not add to it. No aggressive reds unless the verdict is genuinely malicious. No frantic animations. Generous whitespace around the things that matter.
- **Credible** — a professional tool, not a consumer app. Trustworthy like a well-engineered scientific instrument. Users should feel the engineering behind it.

Anti-references: flashy SaaS landing pages, gaming UIs, generic admin dashboards, "cyberpunk" aesthetics with neon cyan-on-dark and purple gradients.

## Aesthetic Direction

**"Analytical instrument."** Think of the refinement of a high-end oscilloscope display, or a well-designed scientific software panel, or a mature developer tool. Utility with craftsmanship. Every element earns its space.

**Theme: dark.** The product's use context is technical investigation, often at night or in a focused work session. Per the impeccable guidance, "observability dashboard for SREs in a dark office → dark" — this is the same class of product.

**Typography (two families, both free on Google Fonts, neither on the reflex-reject list):**
- **Commissioner** — variable-weight geometric sans for all UI text. Distinctive without being trendy, calm without being bland, has real character in its details.
- **Geist Mono** — monospace for hashes, engine names, and any aligned numerical data. Chosen for function (fixed-width is genuinely needed for SHA-256 and per-engine results), not as decorative "tech" shorthand.

**Color (OKLCH throughout):**
- Base surface: deep near-black with a subtle cool tint → `oklch(0.145 0.015 250)`
- Raised surface: `oklch(0.195 0.02 250)`
- Borders: `oklch(0.28 0.015 250)` — low contrast, lets content lead
- Body text: `oklch(0.95 0.008 250)` — off-white, tinted toward base hue
- Muted text: `oklch(0.68 0.018 250)`
- Primary action accent: `oklch(0.78 0.14 65)` — warm amber (unexpected for a security tool — not the cyan cliché)
- Verdict colors:
  - Clean: `oklch(0.72 0.13 155)` — sage green (not electric)
  - Suspicious: `oklch(0.82 0.14 75)` — amber
  - Malicious: `oklch(0.68 0.21 25)` — deep red (not fire-engine bright)

Neutrals are tinted toward the cool base hue (chroma ≈ 0.015–0.02). Accents are used sparingly — the 60/30/10 rule.

## Design Principles

1. **Data is the hero.** Scan results, engine breakdowns, stats — these are why the user came. Give them the most readable typography and the clearest hierarchy. Chrome recedes.
2. **Restraint signals quality.** No gradient text, no left-border accent stripes, no glassmorphism, no sparkline decoration, no card-in-card nesting. Every effect must earn its place.
3. **Rhythm, not uniformity.** Vary spacing for hierarchy. A primary action wants breathing room around it; related labels sit tight. Don't apply the same padding everywhere.
4. **Monospace is functional, not decorative.** Use it for SHA-256 hashes, engine names in tables, and aligned numerical data. Never for body copy or headings as "technical flavor."
5. **Asymmetry where it helps.** Dashboard can have uneven column weights. Upload zone has generous vertical space. Chat pins to the scan context with clear visual anchoring.
6. **Motion conveys state, not delight.** Streaming tokens appear as they arrive (no extra fade). Scan progress uses a calm indeterminate indicator. No bounce, no elastic easing.

## Implementation notes

- Load Commissioner and Geist Mono via `next/font/google` with `variable` output; wire into Tailwind config as `font-sans` and `font-mono`.
- Set CSS variables in `globals.css` using the OKLCH values above; dark class applied at `<html>`.
- Radii: use only `rounded-md` / `rounded-lg`. No `rounded-full` except on true pills and the loading spinner.
- Shadow: one subtle tier (`shadow-sm`) for raised surfaces; no drop-shadows for emphasis.
- Icons: `lucide-react` as specified, rendered at 16px in dense UI, 20px at body scale. Stroke 1.5 for a refined line feel.
