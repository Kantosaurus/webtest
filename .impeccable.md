# Design Context

## Users

The primary audience for this interface is the **CloudsineAI reviewer evaluating this take-home submission**, and any future portfolio viewer who will encounter the repo as a work sample. The product's assignment criteria name *presentation* as an explicit judging axis alongside functionality, code quality, problem-solving, and creativity (see `docs/assignment.md`). That framing changes the target: we are not optimizing for a fictional enterprise buyer or a mass consumer, we are optimizing for a reader who will sit with the interface and judge it as **a piece of craft**.

The product itself still functions — a reviewer will open a file, watch a scan stream in, and chat with the assistant. But the design success metric is not throughput or conversion. It is: *does this feel authored? does every element look like it was decided, not defaulted?*

## Brand Personality

**Three words: authored, composed, unhurried.**

- **Authored** — the interface has a point of view. A reader should feel that someone chose each size, each weight, each margin. No defaults bleed through. No shadcn factory smell. No "AI could have made this."
- **Composed** — a magazine art director's instinct, applied to a product. Type carries hierarchy before color does. Asymmetric rails, deliberate negative space, confident headings. Calm under the tension of the subject matter.
- **Unhurried** — the antidote to the security-tool reflex (alarms, neon, scrolling counters). The UI lets the reader breathe, reads like a considered page, and trusts the content to land. Calm is the single strongest credibility signal this product can emit.

## Aesthetic Direction

**"Publication-meets-product."** The closest references are editorial-sensibility product surfaces: Linear and Vercel marketing pages when they are at their best, Rauno Freiberg's personal site, and modern Anthropic marketing. The work should read as a *tool designed by a publication's art department*. Not longform journalism, not a museum catalog, not an academic figure layout — those are adjacent references to borrow from, but the spine is still a working product.

**Theme: both, user-toggleable.** Ship a considered toggle in the top nav (icon-only, small, placed on the trailing edge). Both themes must be polished to the same bar. The light theme is the *paper* state — warm cream page, cool ink type. The dark theme is the *graphite* state — cool charcoal ground, warm off-white type. Deliberately not a simple inversion: they express different physical materials.

**Typography (three families, each with a defined role):**

- **Bricolage Grotesque** (variable, OFL, Google Fonts) — display and UI sans. Carries headings, nav, section labels, buttons. Has real character across weights and the variable `grade`, `width`, and `XOPQ` axes; reads distinctly without leaning on reflex choices (not Inter, not Plex, not Instrument, not DM).
- **Literata** (variable, OFL, Google Fonts) — editorial serif for body. Used for scan explanations, chat responses, empty-state copy, any paragraph over ~40 words. A contemporary book-face with real warmth. Setting body in a serif is the single most effective editorial cue available for free.
- **Geist Mono** (OFL) — monospace, retained from the current implementation, for SHA-256 hashes, engine names, file sizes, and aligned tabular data. Functional role only; never headings or body as "tech flavor."

Font loading uses `next/font/google` with `variable` output, subset `latin`, and `display: swap`. Wire each to its own CSS variable and expose as `font-sans`, `font-serif`, `font-mono` in Tailwind.

**Color (OKLCH throughout).** Ink + paper + one restrained accent. The accent is **deep ink blue** — a print-magazine signature color, not a security-tool cyan, not the previous amber (warm hues are reserved for the `suspicious` verdict so accent and signal don't collide).

Light theme:
- `--bg`: `oklch(0.980 0.007 85)` — warm cream paper
- `--surface`: `oklch(1 0 0)` — raised (modals, chat bubbles the user sent)
- `--surface-alt`: `oklch(0.965 0.008 85)` — inset panels, code blocks
- `--border`: `oklch(0.90 0.008 85)` — hairline, warm-tinted
- `--ink`: `oklch(0.22 0.015 260)` — cool near-black body
- `--ink-muted`: `oklch(0.48 0.012 260)` — secondary
- `--ink-faint`: `oklch(0.64 0.010 260)` — tertiary / captions
- `--accent`: `oklch(0.42 0.11 260)` — deep ink blue
- `--accent-hover`: `oklch(0.36 0.12 260)`
- `--verdict-clean`: `oklch(0.50 0.09 150)` — sage (not electric)
- `--verdict-suspicious`: `oklch(0.60 0.13 75)` — warm amber

Dark theme:

- `--surface`: `oklch(0.215 0.012 260)` — raised
- `--surface-alt`: `oklch(0.250 0.012 260)` — inset panels
- `--border`: `oklch(0.30 0.010 260)` — hairline

Neutrals in light are tinted warm (hue 85, chroma 0.007–0.012); in dark they tint cool (hue 260, chroma 0.008–0.012). This matches the physical framing — paper is warm, ink is cool; in dark the ground is ink-cool and the type is paper-warm.

Accents appear sparingly — at most one ink-blue element per viewport (the single primary action). Everything else is ink + paper + hairline borders. The 60/30/10 rule applies: ~60% ground, ~30% ink/borders, ~10% accent + verdict signal.

**Anti-references (all four locked in by the user, avoid explicitly):**
- Generic SaaS landing (centered hero, gradient glow, 3-up feature cards, CTA-heavy marketing-page structure)
- Neobrutalist shadcn meme (black borders + yellow + chunky hard drop-shadows, bento-grid playfulness)
- Cyberpunk / crypto security (neon cyan/purple on near-black, glowing borders, terminal theatrics)
- Corporate admin dashboard (side-stripe alerts, identical card grids, blue-gray neutral soup)

## Design Principles

1. **Type carries hierarchy before color does.** Hierarchy is built from size, weight, leading, and placement — in that order. Color is reserved for semantic state (verdict, accent action), not for differentiating between headings and subheadings. If a section needs more prominence, make the type larger or heavier, do not tint it.
2. **The page is a spread, not a canvas.** Compose with a magazine art director's instincts. Asymmetric rails beat centered columns. A scan result can sit left-aligned in a 65–75ch measure with generous right margin and a captioned metadata rail. Empty space is a composition element, not a mistake to be filled.
3. **Body reads in a serif. Chrome reads in a sans.** Literata carries anything over ~40 words (chat responses, explanation text, narrative empty states). Bricolage Grotesque carries UI labels, nav, buttons, headings, section titles. This split is the strongest single editorial cue the interface emits.
4. **Monospace is functional, not decorative.** Only for SHA-256 hashes, engine names in tables, and aligned numeric data (byte counts, detection ratios). Never for body copy, never for headings as "technical shorthand."
5. **Restraint signals quality.** No gradient text, no `border-left: 3px`-style accent stripes, no glassmorphism, no sparkline decoration, no card-in-card nesting, no rounded-rect + generic drop-shadow combination. Shadows should be a single subtle tier or absent entirely. If decoration does not earn its place by supporting content, delete it.
6. **Verdicts are legible without color.** Color-blind safety is a hard requirement. Each verdict chip combines: a distinct word label, a distinct icon (check / triangle / octagon), and a distinct typographic treatment (weight or case). The color is reinforcement, not the primary signal.
7. **Motion conveys state, not delight.** Streaming tokens appear as they arrive with no additional fade. Scan progress uses a calm indeterminate indicator (no bouncing, no elastic easing). Page transitions fade briefly and cleanly. Reserve motion for entrances, exits, and state change — not for decoration.
8. **Contrast aims for AAA where it does not break the aesthetic.** Body copy targets ≥7:1 contrast; large text ≥4.5:1. If a surface cannot reach AAA without flattening the palette (e.g. `ink-faint` captions on cream paper), AA (≥4.5:1 / ≥3:1) is the floor. Never ship below AA.
9. **Both themes are first-class.** Neither theme is a secondary derivation. Light is paper; dark is graphite. The toggle lives in the top nav as a small icon button. Every component is verified in both states before it is considered done.

## Implementation notes

- **Fonts:** load `Bricolage_Grotesque`, `Literata`, and `Geist_Mono` via `next/font/google` in `app/layout.tsx`. Expose as CSS variables `--font-sans`, `--font-serif`, `--font-mono`. Update `tailwind.config.ts` to map `font-sans`, `font-serif`, `font-mono` to those variables.
- **Theme:** switch `<html>` between no-class (light) and `.dark` via a `ThemeProvider` reading `localStorage`, defaulting to `prefers-color-scheme`. Persist the user choice. Toggle component is icon-only (`Sun` / `Moon` from `lucide-react`), ~32×32, in `TopNav`.
- **CSS variables:** rewrite `globals.css` with the two palettes above. Semantic aliases to produce for shadcn compatibility: `--background → --bg`, `--foreground → --ink`, `--muted-foreground → --ink-muted`, `--primary → --accent`, `--primary-foreground` = opposite ground, `--card → --surface`, `--border → --border`. Verdict tokens stay first-class.
- **Radii:** `--radius: 0.375rem` (6px). Use `rounded-md` by default, `rounded-lg` only for the upload dropzone and large panels. No `rounded-full` except true pills (verdict chips) and the loading spinner.
- **Shadows:** one tier. `shadow-sm` for raised surfaces when needed; most surfaces use borders instead.
- **Icons:** `lucide-react` at stroke 1.5, 16px in dense UI, 20px at body scale. Never decorative.
- **Type scale** (fixed `rem` for product UI — fluid `clamp` reserved for any marketing surface if one is added):
  - `text-xs`: 0.75rem / 1rem
  - `text-sm`: 0.875rem / 1.35rem
  - `text-base`: 1rem / 1.6rem (Literata body)
  - `text-lg`: 1.125rem / 1.5rem
  - `text-xl`: 1.375rem / 1.75rem
  - `text-2xl`: 1.75rem / 2.1rem
  - `text-3xl`: 2.25rem / 2.5rem
  - Contrast ratio between adjacent steps ≈ 1.25.
- **Spacing scale** (4pt, semantic tokens): `--space-xs: 4px`, `--space-sm: 8px`, `--space-md: 12px`, `--space-lg: 16px`, `--space-xl: 24px`, `--space-2xl: 32px`, `--space-3xl: 48px`, `--space-4xl: 64px`, `--space-5xl: 96px`.
- **OpenType:** keep `font-feature-settings: 'ss01', 'cv05', 'cv11'` off by default — those stylistic sets are for Commissioner, not Bricolage. Set `font-feature-settings: 'liga', 'calt'` as the baseline; enable `'tnum'` on tabular numeric data (engine counts, byte sizes).
