# 10 · Architecture

The architectural documentation is organised by zoom level, from the world
outside the system down to individual modules.

| Document | Zoom level | Read when |
|---|---|---|
| [System Overview](./system-overview.md) | Narrative | You need the story, not the diagram |
| [Context & Containers](./context-and-containers.md) | C4 L1 + L2 | You want to see what boxes exist and what they talk to |
| [Components](./components.md) | C4 L3 | You want to see what's inside each box |
| [Data Flow](./data-flow.md) | Sequence diagrams | You need to trace a single request end-to-end |
| [Design Decisions](./design-decisions.md) | ADR log | You want to know *why* it was built this way |
